Discussion of the scope of var, let, const

variable scopes in JavaScript:
There are three types of variable scopes in JavaScript- Global scope, Function scope and Block scope.
Global scope: Variables which are declared outside of any function or block have global scope. They can be accessed from anywhere in the code.
Function scope: Variables which are declared inside a function have function scope. They can only be accessed within that function.
Block scope: Variables declared inside a block (with a pair of '{}' braket ,such as an if statement or a for loop) have block scope. They can only be accessed within the block they are declared in.

scope of var, let, const:
The keywords var, let, and const are used to declare variables in JavaScript.
var scope: The scope of a variable declared with var is function scope
let scope: The scope of a variable declared with var is block scope
const scope: The scope of a variable declared with var is block scope


Example:

var a = 3; // Global scoped can be access from any where
let b = 5; // Global scoped can be access from any where
const c = 2; // Global scoped can be access from any where

var d = 7; // function scoped can't be access from outside of this function

if (a < b){
const e=7; // block scoped can't be access from outside of this function
sum = b + e;
}
console.log(sum); //12
}
console.log(sum); // error cause sum is block scoped

Cases of getting null and undefined

The values null and undefined are special values in JavaScript that represent the absence of a value.
null is used to represent the intentional absence of a value. For example, we can assign the value null to a variable to indicate that the variable does not have a value yet.
undefined is used to represent the unintentional absence of a value. For example, a variable that has not been assigned a value has the value undefined.
It is important to note that null and undefined are not the same thing. null is a value, while undefined is not a value. This means that null can be compared to other values, while undefined can't.

What is REST API?

REST stands for Representational State Transfer. This means that when a client requests a resource using a REST API, the server transfers back the current state of the resource in a standardized representation.

The method is the type of request we send to the server. The four main resource methods that are associated with REST APIs are:
GET: This method allows for the server to find the data we requested and sends it back to us.
PUT: If we perform the 'PUT' request, then the server will update an entry in the database.
PATCH: We can perform the 'PATCH' request to partially update an existance resource in the database
POST: This method permits the server to create a new entry in the database.
DELETE: This method allows the server to delete an entry in the database.
